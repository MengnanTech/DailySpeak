## Performance / Playback

- [ ] 优化首次英语语音播放延迟
  - 现象：第一次点击英文朗读时明显偏慢，后续同句播放因为命中客户端 URL 缓存会更快。
  - 当前链路：`/tts/english/mp3 -> 返回 audioUrl -> AVPlayer 播放`，首播延迟可能来自后端 TTS 生成、R2 上传、首次资源下载或播放器预热。
  - 排查目标：区分“后端生成慢”和“拿到 audioUrl 后首播慢”。
  - 可选方向：对高频句子预生成、返回前预热资源、客户端预取当前页英文音频、播放器更早创建 `AVPlayerItem`。
  - 验收：真机首播延迟显著下降，并且控制台日志能明确定位耗时主要落点。
